如何使用Hexo创建自己的博客（零基础）-上篇

发表于 2023-05-25 更新于 2023-05-26 分类于 博客 ， 创作 Valine：
在本篇文章，笔者将利用GitHub加Hexo来帮助读者建立一个属于自己的博客网站，并且给出了安装过程中可能出现问题的一般解决方案，这这一章中主要介绍了利用Git将本地与GitHub库进行链接。

特别声明：本文章主要是借助于这篇知乎大大的文章来进行的，在这里笔者只是稍作整理，用更简练的语言来做了一个流程，以及一些遇到的一些问题和解决方案。

从零开始搭建个人博客（超详细） - 知乎 (zhihu.com)

其实搭建个人博客的过程是最让是捉急的，但完成的那一刻却是让人幸福无比的。

请注意：阅读任何一篇关于安装技术的文章，都应该先把步骤看一遍再进行安装，而不是一步步的对应安装！

本文是利用GitHub和Hexo选择了Next主题来进行搭建的静态网站，总的来说，搭建博客需要以下几步：注册GitHub，创建用于博客的GitHub的库文件，在本地链接到GitHub的库文件，利用GitHub作为我们的服务器来解析博客，下面就进入我们的正题：

注册GitHub

已经拥有GitHub账号的小伙伴可以跳过这一步进行下一步的操作：

  1. 打开GitHub官网：http://github.com/
  2. 点击sign up进行注册，根据官网提示一步步的进行即可；
  3. 需要注意的是在我们选择账户类型的时候选择Free即可，Pro是用于企业或者不想开放的个人用户：
  4. 在Welcome to GitHub界面选择Skip this step即可，这是一个问卷调查，读者可以选择自己感兴趣的内容，也可以不选择跳过；
  5. 验证邮箱通过，至此你就完成了GitHub的注册，请注意务必要记住自己的账户名称，而不是昵称。

GitHub的官网可能有时无法正常访问，这里推荐工具Watt Toolkit，原名为steam++，依据安装提示进行安装即可，勾选GitHub加速即可：

或者使用GitHub520亦可，网上有很多相关教程，再次不多做阐述

本地绑定到GitHub，提交文件测试

我们把GitHub作为我们博客的服务器，因此每次修改文件都需要提交给GitHub，而通过普通的提交往往麻烦且稳定性较差，因此我们需要用Git将本地连接到GitHub上。

链接的方式是SSH（安全外壳协议，Secure Shell 的缩写），无需做深入的理解，可以理解为一个身份认证的功能，只有拥有SSH的钥匙才能证明你的身份，GitHub只有验证了你的身份才会允许你上传文件到对应的库中，是不是类似于你的社交账号和密码，只有你输入密码才能够证实你的身份。
