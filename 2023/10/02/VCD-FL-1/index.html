A.初始化阶段

每个用户$P_i$生成一个随机的行向量$\mathbf{R}_i=\left(r_{i, 1}, r_{i, 2}, \ldots, r_{i, M \cdot\left\lceil\frac{d}{M}\right\rceil}\right)$，用户接收到其他用户发送的全部$\mathbf{R}_i$信息后，通过聚合相加计算出随机的向量$\mathbf{R}$：

我们使用AS所提供的信息计算出对应的 ${f_{[k]}}(x)$信息，当 $x = {a_{(M + 1)k}}$时，该式计算的就是矩阵元素中对应的随机数那一列，但需要注意的是，本式子计算的是一个和式，而并非某一个元素。通过第一个等式我们可以初步判断，AS返回的结果是否符合要求，即验证结果的正确性。

而第二个式子，则通过以下的一个简单推理可以看出 $$ {V_i} = \sum\nolimits_{k = 1}^{\left\lceil {\frac{d}{M}} \right\rceil } {{v_{i,k}}} = \sum\nolimits_{k = 1}^{\left\lceil {\frac{d}{M}} \right\rceil } {{R_{[k]}} \cdot {C_{i,[k]}}} = \sum\nolimits_{k = 1}^{\left\lceil {\frac{d}{M}} \right\rceil } {{R_{[k]}} \cdot (U \cdot {g_{i,[k]}})} = \sum\nolimits_{k = 1}^{\left\lceil {\frac{d}{M}} \right\rceil } {({R_{[k]}} \cdot U) \cdot {g_{i,[k]}}} = \sum\nolimits_{k = 1}^{\left\lceil {\frac{d}{M}} \right\rceil } {{S_{[k]}} \cdot {g_{i,[k]}}} $$
主要使用了承诺以及生成的随机数来帮助我们完成结果的验证，这样可以防止用户与服务器进行串通。

这个地方思考了很久，一部分是因为在最初阅读文献的时候并没有进行认真的阅读； 一部分是因为并没有注意到变量的含义问题，以及矩阵元素的行列关系等等； 因此以后读论文的时候还是要细心，多多注意变量之间的关系！

在常见的双掩码方案中，我们保证系统安全性的前提是任何用户不会同时透露一个用户的线上和线下的份额，然而在本篇文章的假设中，客户可能会与AS进行串通，导致原始的信息被泄露。

本文中的验证过程，当用户存在掉线的情况时，掉线用户的$\mathbb{A}_i$的信息会导致其他用户无法顺利的通过验证，因此我们需要对离线用户的$\mathbb{A}_i$进行恢复，服务器会请求掉线用户的随机种子${\rho _i}$，用来帮助其他用户完成验证的全部过程。
